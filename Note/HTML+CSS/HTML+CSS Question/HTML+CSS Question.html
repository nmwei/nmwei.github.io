2. 如何设置<body>和<div>标签的边框宽度和边框颜色？

答：CSS语法：选择器{border:3px #FF0000 solid}

特别注意：solid不可省。 p、h、img、li标签同样可以设置背景颜色、宽度、边框宽度、边框颜色、内边距、外边距。

注意：在无序列表(<ul><li></li></ul>)中，若设置li的边框宽度(margin-left/right/top/bottom:)、浮动效果(float:)，CSS语法需要作用在li上，而不能作用在ul上。否则，不生效。

设置ul的字号、字体、颜色、加粗等文字样式，li继承；而设置ul 的float:left，li不继承。

<li><a>超级链接文本</a></li> 此时，设置li的字体、字号、颜色，则并不能继承到a，因为a属于链接，有系统默认的字体、字号、颜色。

总结：<ul>的字体设置可以传递到<li>,但不可以传递到<a>，<ul>的浮动效果不可以传递到<li>，<li>的字体设置不可以传递到<a>。

    块级元素的margin、border、padding都可以设置四个方向（top、bottom、left、right四个方向上的宽度/间距。例：（padding-top:10px）。但由于margin和padding属于间距属性，无法设置颜色，只有border可以设置颜色。

3. margin是HTML语法中的属性吗？如何设置文本标签的margin属性值？

答：margin是外边距的意思，属于网页样式CSS语法中定义的内容。不是HTML语法中的属性。与样式有关的属性适宜于用CSS语法定义。注意区分标签在HTML语法中的属性与在CSS语法中的属性。

例如：在HTML语法中，<font face=”” size=”” color=””></font>分别设置文本的字体、字号、颜色。在CSS语法中，{font-family:;font-size:;color:;}分别设置文本样式的字体、字号、颜色。

align是HTML中的语法，text-align是CSS中的语法。由于CSS对于页面样式设置的规范性，text-align更普及。（见上一题）

5. 如何设置图片在块中垂直居中？

错误方法：将图片<img>放在一个<div>中，设置<div>的高度height和行高line-height，且值相同。此时图片的下边缘位于<div>的垂直中央位置，而图片处于<div>的垂直中央位置之上。

(注意：必须设置<div>的height和 line-height，不能通过设置<div>的height和<img />的line-height，与<div>中包裹<p>或<span>不同。<div>中包裹<p>或<span>，既可以通过设置<div>的行高line-height，又可以通过设置<p><span>的行高line-height。见下一题。)

解决方法：①将图片<img>放在一个<div>中，设置<div>的高度height和行高line-height，且值相同。②设置<img />的纵向对齐方式{vertical-align: middle;}。①②两步之后，图片在<div>中完全居中。

11. 当出现给子块级元素（<div/>或者<p/>）设置margin-top,子元素带动父元素整体下移情况时的解决方法：①给父元素加一个边框（border）。②设置父级元素的padding-top=0.1px，使父元素的边框与子元素边框分离。

注意：设置尺寸为像素时，可以设置像素值小于等于1。例如：0.00001px

盒子模型

设置文字在块中居中的错误方法：①设置div的padding-top，这会增加div的内上边距，改变div的高度。②设置div的margin-top，这会改变盒子的位置，使盒子移动。故：设置盒子中文字在盒子中的位置，不可以通过设置盒子的margin和padding的方法，因为改变盒子的margin，会使盒子位置移动，但盒子内文字在盒子的相对位置不变，改变盒子的padding，会使盒子的边框尺寸增大，但盒子中文字在盒子“容积”中的相对位置不变。

特别注意：盒子有三层，内部为实际空间，可“装”块级元素和行内元素，实际空间向外依次为内边距（padding）、盒子边框（border）和外边距（margin）。给盒子<div>设置宽度width和高度height，设置的是盒子的实际空间的宽度（width）和高度（height），并且改变盒子（div）内边距（padding）、盒子边框（border）和外边距（margin）时，盒子的绝对空间并不改变，盒子中内容距离盒子容积边缘的垂直距离和水平距离并不改变，盒子所占据的相对空间增加。

即：设置内边距（padding）、盒子边框（border）和外边距（margin），改变的是盒子的包装厚度，盒子的容积不变。

特别注意：在已设置宽高的div容积中放元素，元素的尺寸可以大于div的容器尺寸，此时元素“溢出”，但是浏览器默认为此容积中元素所占位置仍为div容器尺寸，而不是元素实际尺寸。此时，div内元素与div外其他元素可能发生重合（在内层div设置尺寸范围内，内覆盖外，在内层div设置尺寸范围外，外覆盖内）。

div（其它块级元素同理）若不设置宽度和高度，则默认宽度为100%，默认高度为div内部元素高度。div若设置宽度和高度时，不管内部装什么尺寸的元素，div的实际宽度以设置的宽度和高度为准。例如：若<div><img src=””></div><span>123<span>，若图片尺寸特别大，并没有设置div的宽度，此时div的宽度为100%，高度为图片尺寸，数字123在图片下方一行显示。若设置div的宽度和高度特别小，以设置的宽度和高度为准，数字123与图片发生重合。

13. 超级链接定义伪类a:link{background-color:#}时，浏览器默认为当鼠标移动到该超级链接上时，超级链在水平方向的content和padding都会变色。在垂直方向的content和padding也都会变色。（<a><span>等行内元素没有外边距，内边距只对增加背景颜色的高度起作用）

同理：在盒子模型（块级元素）中，设置盒子的背景颜色{background-color:},背景颜色不仅仅作用在content的范围内，还作用在padding的范围内，但不作用在margin的范围内。

②增大li{margin-left:},点与文字整体向右移动。

  增大li{padding-left:}，点位置不变，文本向右移动。

③将文字与左侧对其的方法：

A：ul{margin-left: Xpx;padding-left: Ypx;}且X+Y=0√

A：ul{margin-left: -40px;}√ X=-40（设置） Y=40（默认）

问题：如何改变边框与点之间的间距？

当图片1与文字2（图片2）在同一行时，设置图片1的vertical-align：middle（默认属性是vertical-align:baseline;），此时与图片1在同一行的文字2（或图片2）不管字号（图片高度）有多大，文字2（或图片2）的水平中心线会比图片1的水平中心线位置稍高。设置图片1的vertical-align：middle属性之后，图片1与同一行图片2的垂直相对位置就基本确定了。

注意：vertical-align:middle/baseline(默认值)/bottom/sub/top/ text-top/ text-bottom/super/百分比%（百分比可为负值）

A：如果设置图片浮动，此时图片与文字不在同一层，图片的vertical-align对文字不发生作用。B：即使同时设置文字的float，图片的vertical-align对文字仍不发生作用。C：如果将文字和图片放在同一个div中，设置div浮动，此时图片与文字在同一层，设置图片的vertical-align对文字发生作用。

D:浮动应理解为盒子尺寸范围内浮动，不会带着溢出部分浮动。因为如果盒子中放图片，盒子尺寸特别小，图片尺寸特别大，盒子外放另一张图片，盒子内图片溢出部分在盒子外图片之下。即使设置盒子浮动，此时盒子外图片移动到这一行，盒子内图片溢出部分仍在盒子外图片之下。（上浮下不浮，浮覆不浮(不浮动的元素必须为块级元素，若为行内元素或行内块元素不会覆盖)，浮盖范围为盒子尺寸，溢出部分被覆盖。）

①不同元素浮动后，这些元素同样不在同一层。但外层元素浮动，其包裹的内层元素在同一层中。

②块级元素独占一行，但此行可以有与此块级元素不在同一层（浮动）的其他元素。

③浮动元素与底层元素在同一位置最多有一个元素。例外：A：内外层盒子嵌套，内层盒子覆盖在外层盒子之上。如果内层盒子中放图片，盒子尺寸特别小，图片尺寸特别大，外层盒子放另一张图片，并设置内层盒子浮动，此时盒子尺寸范围内图片在上层，但图片溢出部分在另一张图片之下。B：上浮下不浮，浮覆不浮。C：div宽高设置的特别小，div中内容溢出，此时溢出的内容可能与盒子外内容重合。

15. margin-left是设置该元素距离同一图层中左侧相邻元素（包括浮动元素和相对定位元素，不包括绝对定位元素）最外层（margin-right）的距离。

float:left是指将该元素浮动到最左侧，浮动的动作有两步，第一步是上浮，第二步是移动到最左侧，将之前在最左侧的内容“挤到”右侧空间。在设置浮动后元素的margin-left属性值时，是指该元素距离左侧相邻元素最外层的距离。

特别注意的是，一个元素所占的位置包括它的包装厚度（margin），而不仅仅是指它的容积。并且<div>属于一个盒子，相当于圈出一块地，如果前后两个<div>，前<div>设置左浮动，则前后盒子重合，并浮动的盒子处于上一层。而如果前后图片，前图片设置左浮动，前后图片并排排列，并且以浮动图片处于上一层（前浮后不浮，浮覆不浮，覆盖的只能是空间，内容被排挤）。

补充：

浮动

基本原则：①元素浮动的特点：a.脱离文档流，浮到上一层，并且只会在同一行内左右浮动，不会垂直方向浮动。（若父元素未设置高度，则其高度由其中子元素撑开，若子元素浮动，则无法继续支撑父元素高度）。b.不管此元素之前是什么类型（行内元素、块级元素、行内块元素），全部转化为行内块元素（有一定的宽高）类型。C.同一行浮动元素，无论高度是否相同，上边框对齐排列。C：自动清除HTML标签中换行引起的元素间距。D:改变html文档流中前后块在页面中同一行的左右顺序。②浮动块元素若不设宽高，浮动块面积等于块中内容面积（其内文本或图片的宽高）（行内块元素在没有设置width和height的情况下，高度和宽度都会自动收缩到能包容下内容的最小高度和宽度。）；浮动块若设置宽高，浮动块面积等于设置的宽高。③浮动块的覆盖与排挤：块浮动后，浮动块面积会与底层块重合（不管底层块有没有设置宽高），并排挤浮动块面积范围内底层文字或图片。

注意：浮动块面积既是覆盖面积又是排挤面积→覆盖的是空间，排挤的是内容（文字或图片）。→当不设宽高时，此面积等于实际内容面积；当设值宽高时，此面积等于设置面积。

元素浮动后，与其他元素（底层元素和浮动元素）在页面中同一行的相对顺序可能改变。（浮动只能左右浮动：一个元素设置浮动之后，只会在同一行内左右移动位置，不会换行。）

注意：左浮动元素从左到右排列，只有当右侧宽度小于下一个左浮动元素的宽度时，下一个左浮动元素才会换行，换成之后成为下一行第一个左浮动元素。换行后，下一行第一个左浮动元素的上边缘与上一行最后一个左浮动元素的下边缘在同一水平线。下一行第一个左浮动元素的左边缘与上一行最后一个左浮动元素左侧第一个比它高的左浮动元素的右边缘在同一竖直线上。同一行所有左浮动元素的上边缘在同一水平线。

俄罗斯方块：①从右上角向左侧落盒子。②一行落满之后才可以下移。

注意：上浮下不浮，浮覆不浮，浮盖范围为盒子尺寸，溢出部分被覆盖。这句话的适用范围如下。同一行浮动元素与浮动元素之间上边缘在同一水平线；浮动元素排挤浮动元素空间及内容；同一行浮动元素与块级元素上边缘处于同一水平线，浮动元素覆盖块级元素空间，排挤块级元素内容；同一行浮动元素与行内块元素上边缘处于同一水平线，浮动元素排挤行内块元素空间及内容；同一行浮动元素与行内元素上边缘处于同一水平线，浮动元素排挤行内元素空间及内容。

 

元素尺寸规则

块级元素的尺寸规则

块级元素在没有设置width和height的情况下，高度和宽度都会自动延伸或收缩。 在标准流中，一个块级元素的盒子水平方向的宽度会自动延伸，直至上一级盒子的限制位置。对于高度，div都是以里面的内容的高度来确定的，也就是会自动收缩到能包容下内容的最小高度。块级元素若设置了width和height，则宽高以设置尺寸为准。

行内块元素的尺寸规则

行内块元素在没有设置width和height的情况下，高度和宽度都会自动收缩到能包容下内容的最小高度和宽度。行内块级元素若设置了width和height，则宽高以设置尺寸为准。

行内元素的尺寸规则

行内元素的高度和宽度都会自动收缩到能包容下内容的最小高度和宽度。行内元素无法设置宽高。

总结：设置固定宽高的块级元素与设置固定宽高的行内块元素特性不同（在换行的相关特性方面表现不同，块级元素即使设置宽高仍占一整行）；没有设置宽高的行内块元素与行内元素特性不同（在内外边距的相关特性方面表现不同）。

同一行静态元素（static）纵向对齐方式

行内块元素与行内块元素在同一行时，行内块元素与行内元素在同一行时，行内元素与行内元素在同一行时，行内块元素中最后一行文字的基线（或图片的底边缘）与行内元素文字的基线（或图片的底边缘）位于同一水平线(vertical-align: baseline)。当行内块元素中没有文字图片等内容时，行内块元素以下边框为基线。

可通过设置vertical-align定义行内元素或行内块元素之间的垂直相对位置。

注意：vertical-align:middle/baseline(默认值)/bottom/sub/top/ text-top/ text-bottom/super/百分比%（百分比可为负值）

①行内块元素的纵向对齐标准线优先级：最后一行文字的基线（或图片的底边缘）＞行内块元素的下边框。

②行内元素的纵向对齐唯一标准线：文字的基线（或图片的底边缘）。

③认为行内块元素中所有内容在同一行，即使行内块元素中内容溢出下边框①②两条仍成立。

④下一行的上边缘位置：A：上一行所有行内块元素的下边框；B：上一行所有行内块元素中最后一行文字的基线（或图片的底边缘）。下一行的上边缘位置与AB中所有位置位于最下方的线位于同一水平线。

对齐：同一行对齐文本优先于边框，换行下方优先于上方。

特别注意：行内块元素在垂直方向溢出部分会增加行高，在水平方向溢出部分（除文字太大和数字字母连续无空格两种情况外水平方向无法溢出）不会增加宽度，会发生覆盖。而块级元素水平方向竖直方向溢出部分都会发生覆盖，不会增加尺寸。

          ②元素2在html文档流中位于浮动元素1之后。

  则清除左浮属性发挥作用，元素2移动到下一行。

  浮动元素1在页面中位于元素2的同一行左侧，浮动元素1在html文档流中位于元素2之前，则浮动元素1一定是左浮。

二、元素2（可以不是浮动元素）定义清除右浮(clear:left)属性，则发挥作用的条件：①元素2在页面中同一行右侧有浮动元素1。

          ②元素2在html文档流中位于浮动元素1之后。

  则清除右浮属性发挥作用，元素2移动到下一行。

  浮动元素1在页面中位于元素2的同一行右侧，浮动元素1在html文档流中位于元素2之前。则浮动元素1一定是右浮。

三、元素2（可以不是浮动元素）定义清除两边浮动(clear:both)属性，意思为clear:left+clear:right。

关键：清除浮动的属性只有作用在html文档流中位置靠后的元素，并与html文档流中位置靠前的浮动元素在同一行发生冲突时，添加清除浮动属性的元素才会发生换行现象，两个条件缺一不可。

注意：清除浮动clear属性不仅可以定义在浮动元素，而且可以定义在底层元素。若作用在浮动元素时，浮动元素（已转化为行内块元素）在浮动前可以是行内元素、行内块元素和块级元素；但若作用在底层元素时，底层元素只能是块级元素，不可以是行级元素，也不可以是行内块元素。

例子：用ul li做水平导航栏。方法：所有的li左浮动，最后增加一个空的li，此li并不浮动，给空的li加清除左浮，ul继续包裹所有li。

过程：li浮动前，与ul在同一文档流，支撑ul高度，而尾li是空的块级元素→li浮动，无法继续支撑ul高度，仅空的尾li支撑ul高度，ul高度坍塌→尾li为空的块元素，被浮动的li覆盖，位于左侧→浮覆不浮，浮仍在左，所有的浮动li认为在空li的左侧→给空li清除左浮动→空li在页面中同一行左侧有浮动li，并且空li在文档流中位于浮动li之后→空li换行→空li换到浮动li高度范围之下的一行（下一行行高为0）→尾li仍撑开ul高度范围→ul包裹两行，涵盖浮动li高度范围。

18. 比较常见的块级元素<div>和行级元素<span>

块级元素会独占一行，默认情况下填满其父元素宽度。行内元素不会独占一行，相邻的行内元素会排列在同一行，直到一行排不下，才会换行，其宽度随元素的内容变化而变化。

块级元素可以设置width、height属性(如未设置，则默认高为0，宽为100%)。行内元素设置width、height属性无效。（<img /> <input />等除外）

块级元素可以设置margin和padding属性。行内元素的margin和padding属性只有水平方向才产生（排斥）效果。<span>和<a>等行内元素在竖直方向完全没有上下外边距margin-top和margin-bottom，但可以设置竖直方向的上下内边距padding-top和padding-bottom，当对行内元素设置竖直方向上下内边距并添加背景颜色时，我们可以看到，行内元素在padding-top、content以及padding-bottom范围内都变色。虽然可以通过设置竖直方向内边距的方式增加行内元素背景颜色的高度，但在上下内边距范围内的颜色与其相邻行元素不发生排斥，可能产生重合的效果，这也是行内元素与块级元素竖直方向内边距的不同之处。

块级元素和行内元素的css相关属性是display。

块级：display:block 行级：display:inline

我们可以通过display属性来切换块级和行内元素。

display:inline-block：行内的块级元素，它拥有块级元素的特点，可以设置长宽，可以设置margin、padding，但是它却不是独占一行，它的宽度并不占满父元素，而是和行内元素一样。

24.将行级元素放在<body>或<div>中，改变浏览器宽度，行级元素的几种不同表现方式：

HTML元素在浏览器展示的方式是：从上到下，从左到右。行内元素是根据浏览器的宽度及自身的width宽度来填充。若当前行剩余空白区域不够，则换至下一行。

基本原则：行级元素放在<body>或<div>（上一层块级父元素）中，在（上一层块级父元素）横向排列由左到右，只有当上一层块级父元素的一行满了之后，才会换行。

①将行级元素放在<body>或<div>中并且不设置<body>或<div>的宽度（默认宽度为父元素宽度的100%），将行级元素放在<body>或<div>中并设置<body>或<div>的宽度为一定百分比%。

此时，<body>或<div>的宽度默认整个屏幕宽度（不设宽度的情况）或整个屏幕宽度（当为<div>时，为父元素宽度）的百分比，随着屏幕宽度的拉缩而变化，其宽度扔占拉伸后屏幕宽度（当为<div>时，为父元素宽度）的原百分比，屏幕下方不出现滚动条。其内的行级元素随着<body>或<div>宽度的变化而自动换行，<body>或<div>的高度自动被撑大（如果之前已经设置了<body>或<div>的高度，文本会溢出。对盒子设置宽度和高度时，既可以设置百分比，又可以设置像素值）。

特殊情况：A：当<body>与<div>中的行级元素为连续的字母，并且字母之间没有空格，此时默认连续的字母为一个文字，即使<body>或<div>的宽度设置的为屏幕的百分比，压缩屏幕宽度，<body>或<div>的宽度扔占拉伸后屏幕宽度的原百分比，但连续的字母仍不自动换行。B：当<body>与<div>中元素为行内块元素inline-block，并且行内块元素的宽度为<body>与<div>的百分比，此时行内块元素随着外层块宽度的压缩而压缩，并不会自动换行。

②将行级元素放在<body>或<div>中并且设置body或<div>的宽度为固定的像素值。

此时，<body>与<div>的宽度默认为固定的像素值，其固定像素值的<body>或<div>宽度不随着屏幕宽度的拉缩而变化，其内的行级元素不随屏幕宽度的拉伸而自动换行，但当屏幕压缩太窄的时候，屏幕下方出现滚动条。

总结：比例固定，可拉伸换行；像素固定，不换行。

28. 元素的定位：

元素的定位

原则：元素的视觉空间不同于元素的占据空间。一般情况下，元素的视觉空间与占据空间重合，元素在此空间内会排挤其他元素。当元素的视觉空间与占据空间分离时，元素的视觉空间会覆盖其他元素，元素的占据空间会排挤其他元素。

①相对定位：{position:relative;top:值;bottom:值;left:值;right:值;}

给元素设置相对定位，当仅设置相对定位属性(position:relative),而不设置top、bottom、left、right值时，该元素视觉空间与占据空间仍重合，都在原位置；当同时设置相对定位属性(position:relative),和top、bottom、left、right值时，该元素的视觉空间与占据空间分离，视觉空间以原位置为基准发生移动，并覆盖移动后位置的其他元素（与浮动float的覆盖不同，浮动仅覆盖块级元素空间，但不覆盖文字图片等内容，而视觉空间会覆盖文本图片等内容），而该元素的占据空间位置不变，仍在原位置。如果想要改变两个相对定位元素的视觉空间的上下层级时，可通过设置元素的Z-index属性值的方式实现。

占据空间的尺寸：content+padding+border+margin。

②绝对定位：{position:absolute;top:值;bottom:值;left:值;right:值;}

当仅设置绝对定位属性（position:absolute），而不设置top、bottom、left、right值时，该元素的占据空间坍塌（消失），但视觉空间仍在原位置(不设置top、bottom、left、right的值与设置值为零不同。前者仍在原位置，后者以body为基准。)当同时设置绝对定位定位属性（position:absolute），和top、bottom、left、right值时，该元素的占据空间坍塌（消失），视觉空间以body左上角为基准发生移动，并覆盖移动后位置的其他元素（与浮动float的覆盖不同，浮动仅覆盖块级元素空间，但不覆盖文字图片等内容，而视觉空间会覆盖文本图片等内容）。如果想要绝对定位元素视觉空间的移动以外层块（不仅限父级）作为基准，可通过设置外层块的相对定位属性(position:relative)的方式实现。如果想要改变两个绝对定位元素的视觉空间的上下层级时，可通过设置元素的Z-index属性值的方式实现。

注意：①一个绝对定位元素的视觉空间与一个相对定位元素的视觉空间仍有上下关系，其中在html文档流中靠后的定位元素位于上。

      ②两个绝对定位元素的视觉空间仍有上下关系，其中在html文档流中靠后的绝对定位元素位于上。

         ③两个相对定位元素的视觉空间仍有上下关系，其中在html文档流中靠后的相对定位元素位于上。

总结：不管绝对还是相对，如果位于同一层（z-index值相同），则文档流（dom(document)树）靠后元素的视觉空间在上。如果想要改变视觉空间的相对上下覆盖关系，可给元素设置Z-index属性（不仅可以作用在绝对定位元素，而且可以作用在相对定位元素，但不可以作用在静态定位元素（position:static）上）。

若绝对定位元素或相对定位元素未设置Z-index属性，系统默认Z-index等于0，虽然静态定位元素的z-index值也为零，但仍被覆盖。即：z-index=X层其实不仅仅是一层，至少有n+1层（随着n的增加层级逐渐增长），从下到上依次为静态定位元素（static）、文档流中顺序从前到后的绝对（相对）元素1、2、3……n。

结论：①静态定位元素的Z-index值系统默认为0（并处于Z-index=0层的最底层，被z-index大于等于0的绝对（相对）定位元素所覆盖，覆盖z-index小于0的绝对（相对）定位元素），设置静态定位元素元素的Z-index值不发生作用。②只有绝对（相对）定位元素设置z-index层级才有效。A：层级大的绝对（相对）定位元素覆盖层级小的绝对（相对）定位元素。B：层级相同时，文档流中靠后的绝对（相对）定位元素覆盖文档流中靠前的绝对（相对）定位元素。

用无序列表做导航栏时，在<li>中包裹<a>,如果要设置li的hover效果，在给li定义间隔时，应通过定义li的宽度，而不能通过定义li的左右边距。因为虽然通过定义li的左右边距也能够达到产生间距的效果，但是当鼠标放在li上时，右侧li块会发生晃动。定义li的宽度时，如果有图标与文字间隔，应给图标与文字分别指定类名，分别定义宽度；如果有竖杠与文字间隔，竖杠由宽度为1px背景颜色为黑色的空li实现。

当采用ul → li → a → div → span嵌套标签的方式设置带有子导航栏的横向导航栏时，如果采用ul{overflow:hidden}的方式清除浮动，此时可以达ul边框包裹li的效果。同时，a会继承ul的{overflow:hidden}属性，起到溢出文本被隐藏的效果，由于div包裹在a中，div是块级元素，a为行内元素，因此div被隐藏。推荐采用给ul设置高的方式实现ul边框包裹左浮动元素li的效果。

34. text-align:center 与 margin:0 auto的对比。

text-align定义在块级（行内块）元素，使块级（行内块）元素中所有的（子级元素、孙级元素…）行内块元素（集合）以及行内元素（集合）在它的父元素中居中。text-align对定义元素中的块级元素不发生作用，但对该块级元素中的子行内块元素或子行内元素仍发生作用。

margin:0 auto只能定义在块级元素，不能定义在行内块元素以及行内元素，使该块级元素在它的父级元素中水平居中。

注意：在IE7.0及之前版本(IE7.0、IE6.0及IE5.5)中，text-align对定义元素中的块级元素仍发生作用；在IE7.0及IE6.0版本中，margin:0 auto定义块级元素在它的父级元素中水平居中也发生作用。

35. 块级元素的外边距既可设置像素值，又可设置百分比，当设置百分比时，以父元素的宽度为基准。例如：margin-top:20%，上外边距为其父元素宽度的20%，margin-left:20%，左外边距为其父元素宽度的20%。

内层块级元素绝对定位，外层块级元素相对定位，设置内层块级元素的top、bottom、left、right值，既可以设置像素值，又可以设置百分比，当设置百分比时，top、bottom百分比值以父级元素高度基准，left、right百分比值以父级元素的宽度为基准。

注意：像素值的设置不仅可以为整数，还可以为小数。
